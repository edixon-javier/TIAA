A) IMPORTANT NOTES - make sure you read this! - top

Before reading forward you should understand how this coverflow works. As you will read installing the coverflow into your html page is straight forward and relative simple. The coverflow is using <ul> and <li> html elements as the base for the categories (the coverflow supports one or more categories), this way if you like you can generate the content with php or some other server side framework using a database or just add the HTML markup manually.

Each thumbnail is displaying an image and optionally display/play a video directly in the thumbnail, or when a thumbnail is clicked a new page can be opened, or open a lightbox that can display various formats like video, audio, Iframe, etc...

If you don't want to use video in the coverflow thumbnails please make sure to set useVideo:"no", the same apply to the lightbox, if you don't want to use the lightbox set useLightbox:"no".

B) Setting up the thumbnails images/html contents - top

Before you start using the coverflow you need to set the thumbnailWidth and thumbnailHeight properties in the coverflow constructor, this two properties represents the thumbnail width and height and make sure you save the images that you want to use at the same width and height with the thumbnailWidth and thumbnailHeight.

The 3D setup is configured such that the center thumbnail will have the actual size, and the ones from the left and right will be scaled down depending on thier position in the 3D space.

If you find it difficult to understand this rules save your images at the same size as we did in our examples.

D) Constructor parameters - top

These parameters represent the possible settings for the coverflow, they are all visible below:

    Main:

  • instanceName:"fwdr3dcov0" - the instance name used to access the API, if you want to use more than one instance please make sure you use the exact name as in the examples fwdr3dcov0 and increase the last number depending of how many coverflow instance you want to use, ex:fwdr3dcov1, fwdr3dcov2, fwdr3dcov3...
  • coverflowHolderDivId:"myDiv" - the div id into which the coverflow will be added (this is also necessary if the displayType is set to fluidwidth).
  • coverflowDataListDivId:"coverflowData" - the id of the <div> element used as the datalist for the coverflow.
  • displayType:"responsive" - the display type, this can be responsive or fluidwidth, all display types are explained and an example is provided for each one of them.
  • coverflowWidth:1600 - This represents the maximum coverflow width in px if the displayType is responsive.
  • coverflowOffsetHeight:120 - This represents the top and bottom gap/padding in px for the coverflow thumbnails.
  • thumbnailResizeOffest:60 - The offset in px plus thumbnail width in px at which the coverflow will start to resize for smaller screens.
  • zIndex:1 - this is a z-index used for the "fluid-width" version of the coverflow so that you can remove conflicts with overlapping menus etc.
  • mainFolderPath:"content" - This represents the main folder path which contains all required files.
  • skinPath:"skin_dark" - this is the skin folder name. This folder contains the coverflow skin for the graphincs/.png skin type.
  • useVectorIcons:no - This can be yes or no, if set to yes the coverflow will use the vector skin otherwise it will use the graphics/.png skin, video tutorial.
  • initializeOnlyWhenVisible:no - this can be yes or no, lazy scrolling / loading, the posibility to initialize the coverflow on scroll when the coverflow is visible in the page, this way for example if the coverflow is in a section of a page that is not visible it will not be initialized, instead the coverflow will be initalized only when the user is scrolling to that section in which the coverflow is added, please note is the display type is fluid width this feature will be ignored.
  • coverflowStartPosition:"center" - this is the thumbnails start position, it can be "left", "right" or "center" or the index of the thumbnail starting from 1.
  • coverflowXRotation:10 - the coverflow rotation on the x-axis.
  • rightClickContextMenu:"developer" - this is modifying the appearance of the context menu when right clicking. If "developer" the context menu will be the developer link "made by FWD", if "disabled" the context menu will be disabled completely, if "default" the context menu will be the browser default.
  • enableMouseWheelScroll:yes - this can be yes or no. Disable or enable the mouse wheel support for scrolling the coverflow.
  • addKeyboardSupport:yes - this can be yes or no, it adds keyboard navigation support for the coverflow, through the left and right arrows.
  • useDragAndSwipe:yes - this can be yes or no. Adds support for drag and swipe with the mouse on desktop machines.
  • backgroundColor:"#0b0b0b" - the coverflow background color.
  • backgroundImage:"" - this coverflow background image path, pelase make sure that the image format is .jpg, .jpeg or .png.
  • backgroundImageRepeat:"no-repeat" - this can be "no-repeat", "repeat", "repeat-x", "repeat-y".
  • backgroundImageSize:"cover" - this can be "auto", "cover", "contain".
  • preloaderBackgroundColor:"#333333" - the coverflow preloader circular animation background HEX color.
  • preloaderFillColor:"#FFFFFF" - the coverflow preloader circular animation fill HEX color.


    • Thumbnails:

  • thumbnailWidth:488 - the width in px of the coverflow thumbnails.
  • thumbnailHeight:320 - the height in px of the coverflow thumbnails.
  • thumbnailXSpace3D:72 - Offset/space in px on the x-axis between the side thumbnails.

    H) API specifications: - top

    The following lists represent all the public API functions and event listeners that you can use from the coverflow externally, the API methods and events are called using the coverflow instanceName, ex: fwdr3dcov0.goToThumb(1);.You can find an example using these API methods and events in the start folder at the api_test.html file.

    Public methods:

    Function Method Description

    isAPIReady

    		 isAPIReady():Boolean Specifies if the API functions are ready to use, possible values are true or false.

    getCurrentCategoryId

    		 getCurrentCategoryId():Number Get the current category id, this method returns a number between 0 and the number of categories - 1.

    switchCategory

    		 switchCategory( Number ):void Set the category id, it will automatically switch to that category just like in the combobox selection. Accepts a parameter that represents a number between 0 and the number of categories - 1.

    getCurrentThumbId

    		 getCurrentThumbId():Number Get the current thumb id, this method returns a number between 0 and the number of thumbs - 1.

    goToThumb

    		 goToThumb( Number ):void Set the thumb id, it will automatically go to that thumb. Accepts a parameter that represents a number between 0 and the number of thumbs - 1.

    isSlideshowPlaying

    		 isSlideshowPlaying():Boolean Specifies if the slideshow is playing, possible values are true or false.

    startSlideshow

    		 startSlideshow():void Starts the coverflow slideshow to play.

    stopSlideshow

    		 stopSlideshow():void Stops the coverflow playing slideshow.

    Event listeners:

    Event Prefix Description

    isAPIReady

    		 FWDR3DCov.IS_API_READY Dispatched when the coverflow instance API is actually available to use.

    categoryChange

    		 FWDR3DCov.CATEGORY_CHANGE Dispatched when the coverflow category has changed. It has the current category id attached to it, you can access it like an attribute "event.id".

    thumbChange

    		 FWDR3DCov.THUMB_CHANGE Dispatched when the coverflow thumb has changed. It has the current thumb id attached to it, you can access it like an attribute "event.id".

    rlShowStart

    		 FWDR3DCov.RL_SHOW_START Dispatched when the lightbox is opened/showed animation has started.

    rlHideComplete

    		 FWDR3DCov.RL_HIDE_COMPLETE Dispatched when the lightbox is hide/close animation is complete.

    goFullScreen

    		 FWDR3DCov.GO_FULLSCREEN Dispatched when the coverflow video player go to full screen mode.

    goNormalScreen

    		 FWDR3DCov.GO_NORMALSCREEN Dispatched when the coverflow video player go to normal screen mode.
